## Architecture Summary

AegisFX is built on four governance principles:

1. **AI advises; deterministic code decides.** OpenAI provides regime analysis. Deterministic Python rules translate that analysis into strategy recommendations and trade proposals.

2. **Two human clicks separate AI from execution.** Operator must explicitly Approve a proposal, then explicitly Execute it. No autonomous trade flow exists.

3. **Every action passes deterministic gates.** Even approved AI proposals must pass risk evaluation, position netting, broker health check, rate limiting, and circuit breaker before reaching the broker.

4. **State persists and reconciles.** SQLite-backed ledger survives restarts. On startup, the system queries the broker for any PENDING trades and resolves them.

### Trade Pipeline (Final)

```
AI proposes
    ↓
PENDING in queue
    ↓ [operator clicks Approve]
APPROVED in queue
    ↓ [operator clicks Execute Trade]
ProposalExecutionBridge
    ↓
TradeOrchestrator.process_trade()
    ↓ (idempotency, circuit breaker, rate limit,
       netting, risk, broker health, trading control)
Broker (OANDA)
    ↓
mark_executed in queue
```
